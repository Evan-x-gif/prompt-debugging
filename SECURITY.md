# 安全说明

## API Key 管理

本项目已移除所有硬编码的 API Key 和敏感配置。

### 配置方式

1. **应用内配置（推荐）**
   - 在应用的设置面板中填入你的 API 配置
   - 配置会保存在浏览器的 localStorage 中
   - 不会被提交到 Git

2. **环境变量配置（可选）**
   - 复制 `.env.example` 为 `.env`
   - 填入你的实际配置
   - `.env` 文件已在 `.gitignore` 中，不会被提交

### 安全最佳实践

- ✅ 使用环境变量或应用内配置管理 API Key
- ✅ 定期轮换 API Key
- ✅ 不要在代码中硬编码敏感信息
- ✅ 不要将 `.env` 文件提交到版本控制
- ✅ 使用 API Key 权限限制（如只读权限）

### 测试文件说明

测试文件（如 `schemas.test.ts`）中使用的 `sk-test-key` 是示例值，不是真实的 API Key，可以安全提交。

### 如果不小心提交了 API Key

1. 立即在 API 提供商处撤销该 Key
2. 生成新的 API Key
3. 使用 `git filter-branch` 或 `BFG Repo-Cleaner` 从 Git 历史中移除敏感信息
4. 强制推送更新后的历史记录

## 报告安全问题

如果发现安全问题，请通过私密方式联系项目维护者，不要公开披露。
